# Prompts — 任務交辦範本

依任務型態選一份範本，把 `<>` 填空補齊後交辦（給 subagent、另一個 agent、或寫進交接）。三要素（目標與動機／驗收條件／回報格式）已內建在範本裡，規則背景見 [`DISPATCH.md`](../DISPATCH.md)。

| 範本 | 用途 |
| --- | --- |
| [`search.md`](search.md) | 找東西：某段程式碼在哪、某個行為由誰觸發、某設定影響哪些地方 |
| [`implement.md`](implement.md) | 實作：有明確 spec 的新功能或修 bug |
| [`refactor.md`](refactor.md) | 重構：行為不變，改結構 |
| [`research.md`](research.md) | 研究：比較方案、查資料、回答開放式問題 |
| [`review.md`](review.md) | 審查：檢查別人（或另一個 agent）的產出 |

通用規則（所有範本共用，不再重複寫在每份裡）：

- 執行者查不到、做不到就**回報「查不到／做不到」＋已嘗試的做法**，不要編造答案硬交差。
- 回報只含：結論、`檔案:行號` 證據、逐條驗收核對。長產物寫檔傳路徑。
- 超出「範圍/邊界」欄位的事一律不做，發現範圍外的問題只回報、不動手。
